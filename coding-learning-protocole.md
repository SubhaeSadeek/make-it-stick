# I. THE PRIME LAW — CODING IS NOT READING, CODING IS RECALL UNDER CONSTRAINT

প্রোগ্রামিং শেখার সবচেয়ে বড় ভুল হলো:

> “আমি বুঝেছি কারণ আমি দেখে বুঝতে পারছি।”

না।

প্রোগ্রামিংয়ে বোঝা মানে:

- না দেখে লিখতে পারা,
- ভুল হলে debug path চিনতে পারা,
- blank screen-এ logic দাঁড় করাতে পারা,
- আগের concept নতুন context-এ টেনে আনতে পারা।

সুতরাং এই protocol-এর মূল আইন:

> **প্রতিটি শেখা session-এ consumption-এর চেয়ে production বেশি হতে হবে।**

Rule:

- 30% input
- 70% output

অর্থাৎ ২০ মিনিট শিখলে অন্তত ৪০–৫০ মিনিট produce করবে।

---

# II. THE FOUR LEARNING STATES OF A CODER

প্রতিটি coding day-তে চারটি state activate করতে হবে:

## 1. Input State

নতুন concept দেখা/পড়া/শোনা।

## 2. Recall State

না দেখে তা পুনর্লিখন।

## 3. Build State

concept ব্যবহার করে ছোট কিছু বানানো।

## 4. Reflect State

কী ভুল হলো, কী gap রইল, কী মনে থাকল।

অধিকাংশ learner প্রথম state-এ আটকে থাকে।
Master learner চার state-ই cycle করে।

---

# III. THE GOLDEN DAILY LEARNING CYCLE (3-HOUR DEEP VERSION)

## BLOCK A — Concept Intake (25 min)

একটি মাত্র micro-topic নিন।
যেমন:

- array methods,
- closures,
- DOM event,
- async flow,
- CSS flex pattern,
- API fetch concept.

শুধু essential বুঝুন। binge নয়।

### Hard Stop Rule:

২৫ মিনিটের বেশি শুধু input নয়।

---

## BLOCK B — Blind Recall Rewrite (20 min)

সব বন্ধ।
YouTube বন্ধ, docs বন্ধ, notes বন্ধ.

এখন:

- syntax লিখুন,
- concept explain করুন,
- ছোট example বানান,
- memory থেকে function লিখুন।

লক্ষ্য: brainকে panic-free struggle-এ পাঠানো।

---

## BLOCK C — Constraint Build Drill (35 min)

এখন micro build:

শর্ত দিয়ে বানান:

- no tutorial,
- no copy,
- limited google,
- one concept must be used.

যেমন DOM event শিখলে:
নিজে todo toggle বানান।

Async শিখলে:
নিজে loading spinner সহ fake fetch বানান।

Constraint makes memory active.

---

## BLOCK D — Error Harvest (15 min)

এখানে coding না। এখানে ভুল সংগ্রহ।

লিখুন:

- কী error এলো?
- কেন এলো?
- আমি প্রথমে কী ভেবেছিলাম?
- আসল কারণ কী?
- next time trigger কী হবে?

ভুল documented না হলে ভুল repeated হয়।

---

## BLOCK E — Retrieval Sprint from Old Topics (20 min)

পুরনো ২–৩ topic থেকে না দেখে:

- function লিখুন,
- definition explain করুন,
- mini challenge করুন।

এটাই spaced strengthening.

---

## BLOCK F — Verbal Reflection (5–10 min)

জোরে বলুন:
আজ আমি কী শিখলাম?
কোথায় আটকালাম?
কাল কী recall করব?

এই verbal closure memory consolidate করে.

---

# IV. THE 7-DAY MEMORY CONSOLIDATION LADDER

প্রতিটি concept এই ladder-এ যাবে:

### Day 0 → Learn + Blind Rewrite

### Day 1 → Recall Again

### Day 3 → Mini Build with Concept

### Day 7 → Mixed Recall + Apply

### Day 14 → Use in Different Project Context

### Day 30 → Teach/Explain Without Help

যে concept এই ladder পার হয়নি, তা mastered না।

# V. THE CORE RETRIEVAL WEAPONS

## 1. Syntax Recall Cards

শুধু definition card নয়।

Front:
"write map() with callback example"

Back:
solution.

## 2. Logic Completion Prompts

"complete this function"
"what comes next"
"find the missing line"

## 3. Explain-this Cues

"Explain event bubbling in plain Bangla"
"Explain closure as if teaching a junior"

## 4. Blank Screen Builds

ফাঁকা VS Code খুলে zero থেকে feature লিখুন।

এটি সবচেয়ে গুরুত্বপূর্ণ।

---

# VI. THE DELIBERATE STRUGGLE PROTOCOL

প্রতিটি coding problem-এ 3 phase:

## Phase 1 — Solo Attempt (15–25 min)

নিজে লড়াই।

## Phase 2 — Targeted Hint (max 5 min)

শুধু stuck point দেখুন। full solution নয়।

## Phase 3 — Rebuild From Scratch

hint দেখে পুরোটা আবার নিজে লিখুন।

Never solution-watch and move on.

---

# VII. THE PROJECT MEMORY ENGINE

শুধু বিচ্ছিন্ন exercise যথেষ্ট নয়।

প্রতি সপ্তাহে:

- 1 micro project
- previous concept reuse mandatory

কারণ প্রকল্প হলো concept collision zone.

যখন পুরনো + নতুন একসাথে লাগে, তখন memory networks knit হয়।

---

# VIII. THE DEBUGGING INTELLIGENCE LOOP

অনেক learner code লিখে, but debugging শেখে না।

প্রতিটি bug-এ follow করুন:

1. Symptom কী?
2. কোথায় break হচ্ছে?
3. Hypothesis 1?
4. Hypothesis 2?
5. Console test কী বলল?
6. Root cause?
7. Prevention rule?

Debugging is reasoning practice, not nuisance.

---

# IX. THE ANTI-TUTORIAL HELL RULES

- consecutive two tutorial days forbidden
- every tutorial must end with blind rebuild
- tutorial clone not enough; mutate feature mandatory
- watch less than you type

আপনি যদি বেশি দেখেন, কম লেখেন — আপনি spectator.

# X. WEEKLY DEEP CODER REVIEW

সপ্তাহে একদিন 60–90 মিনিট:

## Audit 1 — What can I write from memory?

## Audit 2 — Which concepts collapse under blank screen?

## Audit 3 — Which errors repeat?

## Audit 4 — Which topics feel familiar but are not producible?

## Audit 5 — What deserves next week spaced revisit?

এটাই calibration.

# XI. THE MONTHLY CONSOLIDATION RITUAL

মাসে একবার:

- ১টি পুরনো project reopen
- zero help refactor part of it
- same feature cleanerভাবে rewrite
- explain architecture aloud

Nothing reveals growth like revisiting your old code.

# XII. THE ELITE MINDSET RULE

Coding শেখা মানে প্রতিদিন নতুন কিছু দেখা নয়।

Coding শেখা মানে:

> প্রতিদিন অল্প কিছু জিনিসকে এতবার recall, rebuild, apply, debug, explain করা যে সেগুলো আপনার cognitive reflex হয়ে যায়।

Breadth impresses the ego.
Repetition under retrieval builds the engineer.
