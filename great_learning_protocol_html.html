💻 জাভাস্ক্রিপ্ট ও ওয়েব ডেভেলপমেন্ট শেখার জন্য মহাসম্পূর্ণ Learning Protocol

(Make It Stick + Deliberate Practice + Cognitive Mastery ভিত্তিক)

আপনাকে content consumer থেকে active builder-এ, এবং active builder থেকে durable problem-solver-এ রূপান্তর করা।

I. THE PRIME LAW

Coding is not reading, coding is recall under constraint.

  • না দেখে লিখতে পারা
  • ভুল হলে debug path চিনতে পারা
  • blank screen-এ logic দাঁড় করাতে পারা
  • পুরনো concept নতুন context-এ টেনে আনতে পারা

Rule: 30% input, 70% output

II. THE FOUR LEARNING STATES OF A CODER

  • Input State — নতুন concept intake
  • Recall State — না দেখে rewrite
  • Build State — concept দিয়ে build
  • Reflect State — ভুল ও gap review

III. THE GOLDEN DAILY LEARNING CYCLE

  • 25 min Concept Intake
  • 20 min Blind Recall Rewrite
  • 35 min Constraint Build Drill
  • 15 min Error Harvest
  • 20 min Retrieval Sprint from Old Topics
  • 5–10 min Verbal Reflection

IV. THE 7-DAY MEMORY CONSOLIDATION LADDER

  • Day 0 → Learn + Blind Rewrite
  • Day 1 → Recall Again
  • Day 3 → Mini Build
  • Day 7 → Mixed Recall + Apply
  • Day 14 → Different Project Context
  • Day 30 → Teach/Explain Without Help

V. THE CORE RETRIEVAL WEAPONS

  • Syntax Recall Cards
  • Logic Completion Prompts
  • Explain-this Cues
  • Blank Screen Builds

VI. THE DELIBERATE STRUGGLE PROTOCOL

Solo Attempt → Targeted Hint → Rebuild From Scratch

Never solution-watch and move on.

VII. THE PROJECT MEMORY ENGINE

প্রতি সপ্তাহে 1 micro project যেখানে previous concept reuse mandatory.

VIII. THE DEBUGGING INTELLIGENCE LOOP

  • Symptom কী?
  • কোথায় break হচ্ছে?
  • Hypothesis?
  • Console test কী বলল?
  • Root cause?
  • Prevention rule?

IX. THE ANTI-TUTORIAL HELL RULES

  • consecutive two tutorial days forbidden
  • every tutorial must end with blind rebuild
  • tutorial clone not enough; mutate feature mandatory
  • watch less than you type

X. WEEKLY DEEP CODER REVIEW

সপ্তাহে 60–90 মিনিট audit:

  • What can I write from memory?
  • Which concepts collapse?
  • Which errors repeat?
  • What needs spaced revisit?

XI. THE MONTHLY CONSOLIDATION RITUAL

পুরনো project reopen → zero help refactor → architecture explain aloud.

XII. THE ELITE MINDSET RULE

প্রতিদিন অল্প কিছু জিনিসকে এতবার recall, rebuild, apply, debug, explain করা যে সেগুলো cognitive reflex হয়ে যায়।

XIII. WORLD-CLASS DAILY/WEEKLY SNAPSHOT

Daily: 3 hrs deep cycle

Weekly: 5 deep days + 1 project day + 1 audit day

Monthly: refactor + memory audit + blank-screen challenge

বেশি consume নয়, বেশি produce. Blank screen-কে বন্ধু বানাও. Weekly audit ছাড়া improvement invisible.
